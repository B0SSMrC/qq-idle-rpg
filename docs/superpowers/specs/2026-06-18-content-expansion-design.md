# 设计文档：游戏内容中等扩展

- **日期**：2026-06-18
- **状态**：设计完成，待审阅
- **依赖**：`docs/specs/2026-06-18-qq-idle-rpg-design.md`（v1 基础设计）

---

## 1. 概述

在 v1 基础（4 件物品、3 只怪物、20 层深度）上进行中等规模扩展：

- 物品 4 → **37 件**（武器 20 + 护甲 10 + 消耗品 7）
- 怪物 3 → **10 只**，深度 1 → **50 层**
- 新增 **Buff 系统**（临时增益，按步数衰减）
- 体力重平衡 + **体力药水**
- 经济系统联动重设计

## 8. Buff 系统

### 数据模型

```python
@dataclass
class Buff:
    type: str          # "atk" | "def"
    amount: int        # 加成数值
    steps_left: int    # 剩余步数
```

### 规则

- **同类型覆盖**：使用同类型 Buff 药水时替换旧 Buff（不叠加）
- **每步衰减**：探索每成功走一步，所有 Buff 的 `steps_left -= 1`，归零移除
- **探索结束清除**：体力耗尽或战死时，所有 Buff 清空（buff 不跨探索）
- **状态展示**：显示当前 Buff 类型、数值、剩余步数
- **属性计算**：`attack()` / `defense()` 的输入新增 `buffs` 参数，结果 = 基础值 + 装备 + Buff

### DB 表

```sql
CREATE TABLE IF NOT EXISTS buffs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    player_id   INTEGER NOT NULL REFERENCES players(id),
    type        TEXT NOT NULL,        -- "atk" | "def"
    amount      INTEGER NOT NULL,
    steps_left  INTEGER NOT NULL
);
```

读写策略与 `inventory` 一致：读玩家时一并加载，保存时「删旧插新」。

### 旧物品重命名映射

|-------|--------|-------|--------|

## 12. 范围边界（本次不做）

- 武器/护甲/消耗品之外的槽位（戒指、饰品等）
- 战力榜（v2）
- PvP / 组队 / 交易
- 主线剧情
- 装备强化 / 合成 / 分解
- 事件分支选择（玩家主动选而不是纯随机）
